# STM Catalogue Enricher

## When to use

Use this skill when:
- STM markdown files already exist in `stm/output/`
- OpenMetadata contains catalogued table/column metadata for the source schema
- The STMs have empty `Source Table` and `Source Column(s)` in the field-level mapping
- The user wants to complete STMs using the live catalogue

## Prerequisites

- `OPENMETADATA_BASE_URL`, `OPENMETADATA_EMAIL`, `OPENMETADATA_PASSWORD` available in `.env`
- STM files previously generated by `stm-from-data-model`
- OpenMetadata schema already ingested and described
- The OpenMetadata MCP server (`user-openmetadata`) is available

## Orchestration Steps

Follow these steps in order.

### Step 1 — Refresh the catalogue CSV

Run the catalogue export script to keep the flat file artifact up to date:

```bash
set -a && source .env && set +a
python3 .cursor/skills/stm-catalogue-enricher/scripts/enrich_stm_from_catalogue.py \
  --schema-fqn snowflake_fivetran.DRIP_DATA_INTELLIGENCE.BRONZE_ERP__DBO
```

Do NOT read the CSV into context. The subagents will query OpenMetadata directly.

### Step 2 — Discover STM files

Glob `stm/output/*-stm.md` to get the list of all STM files to enrich.

### Step 3 — Launch parallel subagents

For **each** STM file, launch a `generalPurpose` subagent in parallel (all in a single message). Use the `fast` model. Do NOT set `readonly` (it must be false so the agent has MCP access).

Pass each subagent a prompt built from the template below, filling in:
- `{SCHEMA_FQN}` — the OpenMetadata schema FQN (e.g. `snowflake_fivetran.DRIP_DATA_INTELLIGENCE.BRONZE_ERP__DBO`)
- `{STM_FILE_PATH}` — the absolute path to the STM file

#### Subagent prompt template

```
You are enriching a Source-to-Target Mapping (STM) markdown file by querying the OpenMetadata data catalogue.

SCHEMA FQN: {SCHEMA_FQN}
STM FILE TO ENRICH: {STM_FILE_PATH}

## How to query the data catalogue

You MUST use the OpenMetadata MCP tools to look up source metadata. Do not guess column names or data types — always verify against what MCP returns.

**MCP server:** `user-openmetadata`

**Available tools:**

1. `list_tables` — List all tables in a schema with their columns.
   Call: CallMcpTool(server="user-openmetadata", toolName="list_tables", arguments={"database_schema": "{SCHEMA_FQN}"})

2. `get_table` — Fetch one table by FQN including columns, data types, constraints, descriptions, and tags.
   Call: CallMcpTool(server="user-openmetadata", toolName="get_table", arguments={"table_fqn": "{SCHEMA_FQN}.TABLE_NAME"})

3. `get_column` — Fetch one column from a table for a targeted lookup.
   Call: CallMcpTool(server="user-openmetadata", toolName="get_column", arguments={"table_fqn": "{SCHEMA_FQN}.TABLE_NAME", "column_name": "COLUMN_NAME"})

## Query workflow

1. Call `list_tables` with the schema FQN to discover all available source tables and get a summary of their columns.
2. Read the STM file to identify the target table name and all target columns.
3. Reason about which source table(s) are relevant to this target table:
   - Dim tables map to their corresponding operational entity (DimCustomer → CUSTOMERS, DimProduct → PRODUCTS, etc.)
   - Fact tables often span multiple source tables (FactSales → SALES_ORDERS + SALES_ORDER_ITEMS)
   - If no source table maps to the target (e.g. DimDate is generated, DimWarehouse has no source), skip enrichment
4. Call `get_table` for each relevant source table to get full column metadata (names, data types, keys, descriptions).
5. Use the returned metadata to match target columns to source columns and fill the STM.

## Section 7 — Field-Level Mapping Matrix

For each row in the markdown table under "## 7. Field-Level Mapping":

1. If "Source Table" and "Source Column(s)" are already filled, skip the row.

2. If they are empty, find the best matching source column using the MCP data:
   - Use the Target Table name, Target Column name, Field Type, and Description to reason about which source table and column is the correct match.
   - For Foreign Key columns (Field Type = "Foreign Key"), the source column is typically the referenced entity's ID column (e.g. CustomerHashFK → CUSTOMERS.CUSTOMER_ID).
   - For surrogate/hash Primary Key columns (e.g. ProductHashPK, CustomerHashPK), always map to the source table's natural key column (e.g. PRODUCTS.PRODUCT_ID, CUSTOMERS.CUSTOMER_ID). These are NOT ETL-generated with no source — they always derive from a source key.
   - Only match to columns that actually exist in the MCP results. If no reasonable match exists, leave Source Table and Source Column(s) **completely empty** — do NOT write "N/A", "—", "None", or any placeholder. An empty cell means "no source".

3. Fill "Source Table" and "Source Column(s)" with the matched values from MCP.

4. **Write Snowflake SQL expressions** in the "Transformation / Business Rule" column.
   Use the source column names (unqualified, uppercase) directly in the SQL. The downstream
   dbt generator will add table aliases and CTE wrappers.

   CRITICAL: Write real SQL, not English descriptions. Examples:

   | Pattern | SQL Expression |
   |---------|---------------|
   | Hash surrogate key | `CAST(SHA2(COALESCE(CAST(CUSTOMER_ID AS VARCHAR), '#@#@#@#@#'), 256) AS BINARY(32))` |
   | Hash FK lookup | `CAST(SHA2(COALESCE(CAST(ORDER_DATE AS VARCHAR), '#@#@#@#@#'), 256) AS BINARY(32))` |
   | Nullable FK | `IFF(CUSTOMER_ID IS NULL, NULL, CAST(SHA2(COALESCE(CAST(CUSTOMER_ID AS VARCHAR), '#@#@#@#@#'), 256) AS BINARY(32)))` |
   | Type cast | `CAST(UNIT_PRICE AS DECIMAL(19,4))` |
   | Concatenation | `FIRST_NAME \|\| ' ' \|\| LAST_NAME` |
   | Computed measure | `CAST(QUANTITY * UNIT_PRICE AS DECIMAL(19,4))` |
   | Cross-table computation | `CAST(QUANTITY * COST_PRICE AS DECIMAL(19,4))` (note: list both source tables) |
   | Date extraction | `CAST(CREATED_AT AS DATE)` |
   | Boolean derivation | `NOT IS_ACTIVE` or `IFF(STATUS = 'Active', TRUE, FALSE)` |
   | String cleanup | `TRIM(NAME)` |
   | Direct passthrough (same type) | (leave Transformation empty — no transform needed) |

   **NEVER use LEFT() to truncate columns.** The target data type lengths from the data model
   are tentative — source system precision takes precedence. If the source column is
   VARCHAR(255) and the target model says VARCHAR(50), do NOT truncate. Let the source
   precision flow through. TRIM() is acceptable for whitespace cleanup, but do not wrap
   columns in LEFT(..., N) to force them into data model lengths.

   If the source data type matches the target type exactly, leave Transformation **empty** —
   no cast or transform is needed for a direct passthrough.

   If "Transformation / Business Rule" already has content, append rather than replace.

5. For audit/metadata columns (EtlBatchId, LoadTimestamp) and any column with no source match, leave Source Table and Source Column(s) **completely empty** — do NOT write "N/A", "—", "None", or any placeholder. These cells must be blank. Only write a Source Table value when a real source table name from MCP matches.

6. For derived/computed columns that combine multiple source columns (e.g. FullName from FIRST_NAME + LAST_NAME, GrossAmount from QUANTITY × UNIT_PRICE):
   - Fill Source Table with the table the inputs come from.
   - Fill Source Column(s) with all input columns separated by commas (e.g. "FIRST_NAME, LAST_NAME").
   - Write the SQL expression in Transformation (e.g. `FIRST_NAME || ' ' || LAST_NAME`, `CAST(QUANTITY * UNIT_PRICE AS DECIMAL(19,4))`).
   - The key distinction: if the column's value can be traced back to source columns, list them. Only leave source empty for truly ETL-generated values with no source input (batch IDs, load timestamps, SCD dates).

## Section 8 — Load Strategy

If the Load Strategy table has an empty first data row:
- Check the MCP results for UPDATED_AT, MODIFIED_AT, or LAST_MODIFIED columns on the matched source tables.
- If found: Load Type = "Incremental", Method = "Delta load using <table>.<column>".
- If not found: Load Type = "Full", Method = "Full table refresh".
- For date dimensions or reference tables with no source: Load Type = "Full", Method = "Generated date spine — one-time load" or "Manual/reference data load".

## General rules

- You MUST query MCP before filling any source fields. Never guess.
- Only fill fields derivable from the catalogue. Never invent source columns.
- Preserve all existing non-empty values — only fill blanks.
- Use StrReplace for targeted edits. Do NOT rewrite the entire file.
- Return a brief summary: how many rows matched, which source tables used, any tables skipped.
```

### Step 4 — Post-enrich: classification tags & glossary terms

After all subagents complete, run the deterministic post-enrichment script.
This reads each STM's Section 7 to find matched source pairs, fetches tags and glossary
term assignments from OpenMetadata once per unique source table, and fills Sections 5 and 6.

```bash
set -a && source .env && set +a
python3 .cursor/skills/stm-catalogue-enricher/scripts/post_enrich_tags_glossary.py \
  --schema-fqn snowflake_fivetran.DRIP_DATA_INTELLIGENCE.BRONZE_ERP__DBO
```

This step is **not** done by the subagents — it runs once at the end because
tag/glossary lookup is a mechanical operation that only needs the matched source
pairs (already written by the subagents) plus a small number of deduplicated API calls.

### Step 5 — Confirm results

After the post-enrichment script completes, report to the user:
- How many STMs were enriched vs skipped
- Which source tables were mapped to which target tables
- Classification tags and glossary terms populated
- Any STMs that could not be enriched (e.g. DimDate with no source table)

## Population rules

- Only fill fields that can be derived from the catalogue; leave others blank.
- Never invent source columns, owners, thresholds, or orchestration tools.
- Preserve all existing non-empty values in the STM — only fill blanks.
- If a target column cannot be matched in the catalogue, leave source fields empty.
- The Data Type column represents the TARGET type. Do not overwrite it with source types.
- **Never truncate with LEFT().** Source system column precision takes precedence over target data model lengths. Use TRIM() for whitespace only.
