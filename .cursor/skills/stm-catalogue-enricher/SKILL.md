# STM Catalogue Enricher

## When to use

Use this skill when:
- STM markdown files already exist in `stm/output`
- OpenMetadata contains catalogued table/column metadata for the same schema
- The STMs have empty `Source Table` and `Source Column(s)` in the field-level mapping
- The user wants to complete STMs using the live catalogue

## Prerequisites

- `OPENMETADATA_BASE_URL`, `OPENMETADATA_EMAIL`, `OPENMETADATA_PASSWORD` available in `.env`
- STM files previously generated by `stm-from-data-model`
- OpenMetadata schema already ingested and described

## Run

```bash
set -a && source .env && set +a
python3 .cursor/skills/stm-catalogue-enricher/scripts/enrich_stm_from_catalogue.py \
  --stm-dir stm/output \
  --schema-fqn snowflake_fivetran.DRIP_DATA_INTELLIGENCE.BRONZE_ERP__DBO
```

Both flags are optional and default to the values shown above.

## What it does

1. Connects to OpenMetadata and pulls all table/column metadata for the target schema
2. Builds an in-memory data catalogue (same shape as `.cursor/flat/data_catalogue.csv`)
3. Reads each STM markdown file in `stm-dir`
4. For each row in the **Field-Level Mapping Matrix** (section 7):
   - Matches `Target Column` → catalogue `column_name` (case-insensitive)
   - Fills `Source Table` with the catalogue `table_name`
   - Fills `Source Column(s)` with the catalogue `column_name`
   - Appends source data type as a transformation note when it differs from target type
5. Populates a basic **Data Quality & Validation Rules** scaffold (section 9):
   - NOT NULL check for non-nullable key columns
   - Referential integrity stub for foreign-key columns
6. Populates a basic **Load Strategy** scaffold (section 10):
   - Incremental / full-load hint based on analyzer incremental columns
7. Writes the enriched STM back to the same file
8. Exports the catalogue to `.cursor/flat/data_catalogue.csv`

## Population rules

- Only fill fields that can be derived from the catalogue; leave others blank.
- Never invent source columns, owners, thresholds, or orchestration tools.
- Preserve all existing non-empty values in the STM — only fill blanks.
- If a target column cannot be matched in the catalogue, leave source fields empty.
